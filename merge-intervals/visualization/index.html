Merge Intervals

You're given a list of closed intervals [start, end]. Whenever two of them overlap, fuse them into one — and return the smallest set of intervals that still covers everything in the input.

1 ≤ intervals.length ≤ 10³ 0 ≤ start ≤ end ≤ 10⁴
Today's example [8,10]  [1,3]  [2,6]  [15,18] → [1,6]  [8,10]  [15,18]
Step 1 — Sort by start time

Overlaps only become
visible once the order is right

An overlap could be hiding anywhere in an unsorted list — checking every pair would cost O(n²). Sort by start time first, and any interval that could overlap the one we're building sits right next to it.

Given order

def merge(intervals):
    ordered = sorted(intervals,
                      key=lambda iv: iv[0])
 
    merged = [ordered[0]]
 
for start, end in ordered[1:]:
        last = merged[-1]
        if start <= last[1]:
            last[1] = max(last[1], end)
        else:
            merged.append([start, end])
 
return merged

Python · Complete Solution

The full merge() function

from typing import List
 
def merge(intervals: List[List[int]]) → List[List[int]]:
    if not intervals:
        return []
 
    ordered = sorted(intervals, key=lambda iv: iv[0])
    merged = [ordered[0][:]]
 
    for start, end in ordered[1:]:
        last = merged[-1]
        if start <= last[1]:
            last[1] = max(last[1], end)
        else:
            merged.append([start, end])
 
    return merged

O(n log n) — sort dominates; sweep is O(n)
Space
O(n) — sorted copy + merged output
